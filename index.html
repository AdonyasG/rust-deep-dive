🎯 Strings — First Real Gotcha
✓
String = owned, heap-allocated, mutable — you own this data
✓
&str = a borrowed slice, a window into existing string data
✓
Convert between them: .to_string(), String::from(), .as_str()
✓
String concatenation works differently — can't just + two Strings freely
let owned: String = String::from("hello"); let borrowed: &str = "world"; // lives in binary let also_borrowed: &str = &owned; // slice of String above
rust
🐹Go's string is immutable and passed by value always. Rust forces you to decide — do you OWN this string or just LOOK at it? That question is the seed of all of Phase A.

0%
▾

🎯 Ownership Rules — Learn These Cold

There are only 3 rules. Everything else is a consequence of them:

✓
Rule 1: Every value has exactly one owner
✓
Rule 2: When the owner goes out of scope, the value is dropped (freed)
✓
Rule 3: There can only be one owner at a time
let s1 = String::from("hello"); let s2 = s1; // s1 is MOVED into s2 — s1 no longer exists println!("{}", s1); // 💥 compiler error — s1 was moved
rust
🐹In Go this works fine because the GC tracks references. In Rust there's no GC — the compiler enforces single ownership so it knows exactly when to free memory
🎯 Copy vs Move
✓
Primitives (i32, bool, f64, char) are copied — assignment duplicates them
✓
Heap types (String, Vec) are moved — assignment transfers ownership
✓
Understand why let s2 = s1 kills s1 but let b = a where a: i32 doesn't
let x: i32 = 5; let y = x; // copied — x still valid println!("{}", x); // ✅ fine let s1 = String::from("hi"); let s2 = s1; // moved — s1 gone println!("{}", s1); // 💥 error
rust
🎯 References & Borrowing
✓
Use & to borrow a value without taking ownership
✓
The original owner keeps ownership — you just get a read-only view
✓
Multiple immutable references are fine simultaneously
fn print_length(s: &String) { println!("{}", s.len()); } let s = String::from("hello"); print_length(&s); // borrow s println!("{}", s); // s still valid — we only borrowed it
rust
🐹Like passing *string in Go, except Go trusts you not to misuse it. Rust's compiler verifies it at compile time
🎯 Mutable References
✓
Add mut to allow the borrower to modify the value
✓
Golden rule: one mutable reference OR many immutable ones — never both
✓
Understand how this rule eliminates data races at compile time
let mut s = String::from("hello"); let r1 = &s; // immutable borrow let r2 = &s; // another immutable borrow — fine // let r3 = &mut s; // 💥 can't mix mutable + immutable let r3 = &mut s; // ok after r1, r2 are no longer used r3.push_str(" world");
rust
🎯 Borrow Checker Errors — Lean Into Them
✓
Deliberately write code that violates ownership rules
✓
Read every error message fully — Rust's errors are unusually helpful
✓
Trigger: use a variable after moving it
✓
Trigger: hold an immutable and mutable borrow at the same time
✓
Trigger: return a reference to a local variable
🎯 clone() — When and Why to Avoid It
✓
clone() makes a deep copy — both variables are now independent owners
✓
It always works, but it's often a crutch that hides a misunderstanding
✓
If you're cloning to fix a borrow error, ask why first
💡Rule of thumb: if you clone more than once or twice in a small program, you probably haven't understood ownership yet. Treat clone() as a code smell until you can explain why you need it.

🎯 Structs & Methods
✓
Define data with struct — identical concept to Go
✓
Add behavior with impl blocks — like Go methods on types
✓
&self = read-only method, &mut self = mutating method, self = consumes the value
struct Server { host: String, port: u16, } impl Server { fn address(&self) -> String { format!("{}:{}", self.host, self.port) } }
rust
🐹Exactly like func (s Server) Address() string — just different syntax
🎯 Enums & Match
✓
Rust enum carries data per variant — far more powerful than Go's iota
✓
match is exhaustive — compiler forces you to handle every variant
✓
No default escape hatch unless you use _ explicitly
enum Status { Running, Failed(String), // carries a message Restarting(u32), // carries retry count } match status { Status::Running => println!("ok"), Status::Failed(msg) => println!("error: {}", msg), Status::Restarting(n) => println!("retry {}", n), }
rust
🐹In Go you'd use a struct + constants + type assertions. Rust collapses all of that into one clean enum
🎯 Option<T> — Rust's Answer to nil
✓
No null in Rust — absence is represented as Option<T>
✓
Some(value) = there's a value, None = there isn't
✓
Compiler forces you to handle both cases — no nil panics ever
🐹Like returning (value, bool) in Go — but the compiler won't let you ignore the None case
🎯 Result<T, E> & The ? Operator
✓
Result<T, E> = Ok(value) or Err(error) — maps directly to Go's (T, error)
✓
The ? operator: if Err, return it early — if Ok, unwrap and continue
✓
This replaces Go's if err != nil { return err } pattern
fn read_config(path: &str) -> Result<String, std::io::Error> { let content = std::fs::read_to_string(path)?; // ? returns early on error Ok(content) }
rust
🐹? is exactly if err != nil { return nil, err } — one character instead of three lines
🎯 Vec & HashMap
✓
Vec<T> = Go slice — dynamic, heap-allocated, ordered
✓
HashMap<K, V> = Go map — same concept, same usage
✓
Both follow ownership rules — pass by value or reference accordingly
🎯 Iterators — .map(), .filter(), .collect()
✓
Rust iterators are lazy — nothing runs until .collect() or similar
✓
Chains read like a pipeline — familiar if you've used functional style
let ports: Vec<u16> = vec![80, 443, 8080, 3000]; let high_ports: Vec<u16> = ports .iter() .filter(|&&p| p > 1024) .copied() .collect();
rust
🎯 Traits
✓
A trait defines behavior a type must implement — like a Go interface
✓
Go: interface satisfaction is implicit. Rust: explicitly write impl Trait for Type
✓
Traits can have default implementations — more powerful than Go interfaces
🐹Same idea as Go interfaces — just swap implicit for explicit
🎯 Lifetimes — Recognize the Syntax
✓
Lifetimes tell the compiler how long a reference is valid
✓
You'll see 'a annotations in function signatures with multiple references
✓
Don't go deep here — just don't panic when you see it
// just recognize this shape — don't memorize it yet fn longest<'a>(x: &'a str, y: &'a str) -> &'a str { if x.len() > y.len() { x } else { y } }
rust
🎯 Code Organization with mod
✓
mod = Go package — groups related code
✓
Split into multiple files naturally as your project grows
✓
pub = exported (like Go's capitalized names)
